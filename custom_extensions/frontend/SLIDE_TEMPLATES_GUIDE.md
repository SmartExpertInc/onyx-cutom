# Professional Slide Templates Guide for AI Assistant

## Overview
This guide provides instructions for the AI assistant on how to create professional presentations using 10 specialized slide templates. Each template is designed for specific content types and follows industry best practices from Google Slides, PowerPoint, and Gamma.

## Template Specifications

### 1. Title Slide (`layout-title`)
**Purpose**: Opening slide for presentations
**Icon**: ◯
**Layout**: Centered content with large typography
**Content Structure**:
- Main title (Headline Level 1)
- Subtitle or key message (Headline Level 2)
**Usage**: Use for presentation opening, section dividers, or major announcements
**Example**:
```
Title: "Digital Transformation Strategy"
Subtitle: "Roadmap for 2025"
```

### 2. Content Slide (`layout-content`)
**Purpose**: Standard content delivery with title and body
**Icon**: ▫
**Layout**: Traditional slide layout with title and content area
**Content Structure**:
- Slide title (Headline Level 1)
- Main content paragraph (comprehensive text)
**Usage**: Detailed explanations, background information, or narrative content
**Example**:
```
Title: "Market Analysis Overview"
Content: "Our comprehensive market research reveals three key trends..."
```

### 3. Four Key Points (`layout-four-bullets`)
**Purpose**: Exactly 4 key points in grid formation
**Icon**: ▪
**Layout**: 2x2 grid layout for bullet points
**Content Structure**:
- Title (Headline Level 1)
- Exactly 4 bullet points arranged in grid
**Usage**: When you have exactly 4 main points, benefits, features, or key items
**Example**:
```
Title: "Core Benefits"
Bullets: ["Increased Efficiency", "Cost Reduction", "Better Quality", "Faster Delivery"]
```

### 4. Six Key Points (`layout-six-bullets`)
**Purpose**: 6 bullet points in two-column layout
**Icon**: ▫
**Layout**: Two columns with 3 points each
**Content Structure**:
- Title (Headline Level 1)
- Exactly 6 bullet points in two columns
**Usage**: Comprehensive lists, feature comparisons, or detailed breakdowns
**Example**:
```
Title: "Implementation Steps"
Bullets: ["Analysis", "Planning", "Development", "Testing", "Deployment", "Monitoring"]
```

### 5. Process Steps (`layout-steps`)
**Purpose**: Sequential numbered process or workflow
**Icon**: ◐
**Layout**: Two-column grid for numbered steps
**Content Structure**:
- Title (Headline Level 1)
- Numbered list (4+ items work best)
**Usage**: Workflows, procedures, methodologies, or sequential processes
**Example**:
```
Title: "Project Methodology"
Steps: ["Discovery Phase", "Design Phase", "Development Phase", "Testing Phase"]
```

### 6. Two Column (`layout-split`)
**Purpose**: Side-by-side content comparison or information
**Icon**: ◑
**Layout**: Equal two-column layout
**Content Structure**:
- Main title (Headline Level 1)
- Left column header (Headline Level 2)
- Left column content (Paragraph)
- Right column header (Headline Level 2)
- Right column content (Paragraph)
**Usage**: Comparisons, before/after, pros/cons, or parallel information
**Example**:
```
Title: "Technology Comparison"
Left: "Current System" + description
Right: "Proposed System" + description
```

### 7. Comparison (`layout-comparison`)
**Purpose**: Before/after or direct comparison analysis
**Icon**: ◒
**Layout**: Two-column comparison with emphasis on differences
**Content Structure**:
- Main title with "Comparison" in the name (Headline Level 1)
- Before/Old section (Headline Level 2 + Paragraph)
- After/New section (Headline Level 2 + Paragraph)
**Usage**: Before/after scenarios, old vs new, problem vs solution
**Example**:
```
Title: "Performance Comparison Analysis"
Before: "Current Performance" + metrics
After: "Expected Performance" + improved metrics
```

### 8. Quote/Highlight (`layout-quote`)
**Purpose**: Emphasize important quotes or key messages
**Icon**: ◓
**Layout**: Centered, prominent text display
**Content Structure**:
- Title (Headline Level 1)
- Quote or key message (Paragraph with quotation marks)
**Usage**: Customer testimonials, important quotes, mission statements, or key messages
**Example**:
```
Title: "Customer Feedback"
Quote: "This solution transformed our business operations completely."
```

### 9. Agenda (`layout-agenda`)
**Purpose**: Meeting or presentation agenda
**Icon**: ◔
**Layout**: Clear numbered list format
**Content Structure**:
- "Agenda" as title (Headline Level 1)
- Numbered list of agenda items
**Usage**: Meeting agendas, presentation outlines, or event schedules
**Example**:
```
Title: "Agenda"
Items: ["Welcome & Introductions", "Project Overview", "Q&A Session", "Next Steps"]
```

### 10. Summary (`layout-summary`)
**Purpose**: Key takeaways and conclusion
**Icon**: ◕
**Layout**: Takeaways list with closing statement
**Content Structure**:
- Title with "Summary", "Takeaways", or "Conclusion" (Headline Level 1)
- Key takeaways (Bullet list)
- Closing statement (Paragraph)
**Usage**: Presentation conclusions, meeting summaries, or key takeaways
**Example**:
```
Title: "Key Takeaways"
Bullets: ["Budget approved", "Timeline confirmed", "Team assigned"]
Closing: "Thank you for your attention. Questions?"
```

## AI Assistant Guidelines

### Template Selection Rules
1. **Analyze content first**: Look at the number and type of content elements
2. **Match to purpose**: Select template based on the slide's primary purpose
3. **Consider flow**: Use title slides for section breaks, summary for conclusions
4. **Optimize for readability**: Choose layouts that best present the information

### Content Optimization
1. **Keep text concise**: Each bullet point should be 3-8 words maximum
2. **Use parallel structure**: Maintain consistent grammar and format
3. **Prioritize clarity**: Ensure content fits the template's visual hierarchy
4. **Balance information**: Don't overcrowd slides with too much content

### Template Triggers
- **4 items exactly** → Four Key Points template
- **6 items exactly** → Six Key Points template
- **Sequential process** → Process Steps template
- **Side-by-side info** → Two Column template
- **Before/after** → Comparison template
- **Important quote** → Quote/Highlight template
- **Meeting structure** → Agenda template
- **Presentation end** → Summary template
- **Section start** → Title template
- **Everything else** → Content template

### Best Practices
1. **Maintain consistency**: Use similar templates for similar content types
2. **Visual hierarchy**: Ensure titles are clear and content supports them
3. **Professional language**: Use business-appropriate, clear language
4. **Logical flow**: Arrange slides in a logical sequence
5. **Call to action**: End with clear next steps or questions

## Implementation Notes
- Templates auto-detect based on content structure and slide titles
- Layout classes are applied automatically based on content analysis
- All templates support inline editing without additional UI elements
- Templates are responsive and work across all screen sizes
- Icons use outline style for professional appearance
